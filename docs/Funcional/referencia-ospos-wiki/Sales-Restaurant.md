The application has a "Table" feature inside the **Sale** module for restaurant use. For other module documentation, see [Getting Started Usage](Getting-Started-usage).

Complete module list at [Complete Feature Datasheet](Complete-feature-datasheet#complete-list-of-features)

## Restaurant Tables

Clients are grouped in "Tables", acting as labels for transactions. You can sell to a table with or without a client - ideal for meeting combinations.

## How to Use

1. First, enable Restaurant Tables in the **Office** module.
2. A new selector box will appear in the **Sale** module, next to the Sale Mode.
3. Select the Table.
4. Load the items and suspend the sale to start with a new table.
5. When you suspend the transaction, the table disappears from your select list.
6. To resume another table sale, choose that table and go to the suspended button.
7. Unsuspend the table's sale to keep adding items and/or complete the sale.

> **Note:** This workflow is counter-intuitive but functional. Future improvements are planned to make it more user-friendly. Contributions welcome!
